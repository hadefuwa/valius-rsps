Developer Commands

All debug and tooling commands available in-game. Most require Owner rank; some work at Admin level — each card notes the minimum rank.

Owner/Admin only. These commands are not available to regular players. Type them in chat prefixed with ::. All output goes to your chat window unless otherwise noted.
🔍

📍 Position & Navigation 4 commands

coords ⚙ Admin
Print your current tile coordinates to chat.
Syntax::coords
OutputChat message with X, Y, and height (plane).
Sourcecommands/admin/Coords.java
Example
coords
Your coordinates are: 3090, 3500 0
pos 👑 Owner
Verbose position output — same as ::coords with slightly different format.
Syntax::pos
OutputChat message with labelled x, y, h values.
Sourcecommands/owner/Pos.java
Example
pos
Current coordinates x: 3090 y: 3500 h: 0
tele [x] [y] [z] ⚙ Admin
Teleport yourself to any tile on the map by coordinates.
Syntax::tele <x> <y> <z>
Argsx — tile X   y — tile Y   z — height plane (0 = ground)
Sourcecommands/admin/Tele.java
Examples
tele 3090 3500 0
Teleport to home area, ground floor
tele 3222 3219 0
Teleport to Lumbridge
tele 2452 9481 0
Teleport to Fight Cave area (underground)
chunkinfo 👑 Owner
Print your position within the current map chunk.
Syntax::chunkinfo
OutputYour X and Y offset within the 8×8 chunk grid.
SourceInline in Commands.java
Example
chunkinfo
X: 2, Y: 4
Position within the current 8x8 chunk

🔎 ID Lookup 2 commands

getid [item name] 👑 Owner
Search the item database by name and return matching item IDs.
Syntax::getid <name>
ArgsMinimum 3 characters. Case-insensitive. Returns up to 100 results.
OutputList of matching items as ItemName - ItemID, then a result count.
Sourcecommands/owner/Getid.java
Examples
getid abyssal whip
Searching: abyssal whip
Abyssal whip - 4151
Abyssal whip (or) - 12773
2 results found...
getid dragon
Dragon scimitar - 4587
Dragon dagger - 1215
Dragon sword - 21009
... (up to 100 results)
Only searches items. Use ::npc to test-spawn an NPC if you need to find its ID by appearance.
shop [shop-id] 👑 Owner
Open any shop directly by its numeric ID — useful for inspecting shop contents without finding the NPC.
Syntax::shop <id>
Sourcecommands/owner/Shop.java
Examples
shop 119
Opens the Blood Money shop (tab 4 of Point Reward Shops)
shop 125
Opens the PVM Points shop
shop 2
Opens the General Store
Shop IDs are defined in Data/json/shops/points.json and the other shop JSON files. The TabbedShop enum maps category names to their IDs.

🔄 Hot-Reload 11 options

reload [subsystem] 👑 Owner
Hot-reload server data without restarting. Pass a subsystem name as the argument.
Syntax::reload <subsystem>
Sourcecommands/owner/Reload.java
All available subsystems
reload shops
@blu@Reloaded Shops.
Re-reads all shop JSON files from disk
reload drops
@blu@Reloaded Drops.
Re-reads NPC drop tables and RNG data
reload items
@blu@Reloaded Items.
Re-reads item config, definitions, and prices
reload npcs
@blu@Reloaded Npcs.
Re-reads NPC spawns and handler data
reload doors
@blu@Reloaded Doors.
Re-reads door/object definitions
reload objects
@blu@Reloaded Objects.
reload punishments
@blu@Reloaded Punishments.
Re-reads ban and mute lists from disk
reload commands
@blu@Reloaded Commands.
Clears the reflection command class cache — forces re-lookup next use
reload looting
@blu@Reloaded Looting.
Toggles Config.BAG_AND_POUCH_PERMITTED
reload tp
@blu@Reloaded Tp.
Saves then reloads trading post listings
Use ::reload commands after adding a new command class to pick it up without a full server restart. Other reloads are safe to run on a live server.
No validation before reload — a malformed JSON file will crash the subsystem. Always check your JSON before reloading.
reload (no args) ⚙ Admin
Admin-level reload — clears and reinitialises the combat script handler.
Syntax::reload (no argument)
OutputChat message: Scripts reloaded.
Sourcecommands/admin/Reload.java
Example
reload
Scripts reloaded.

🖥 Server State 3 commands

serverstatus 👑 Owner
Print server thread pool status to the console.
Syntax::serverstatus
OutputChat: "See console for server status information." — full output in the server console window.
Sourcecommands/owner/Serverstatus.java
Console output
IO_THREAD
Shutdown? false
Terminated? false
myplayers 👑 Owner
List all currently connected player session indices to the console.
Syntax::myplayers
OutputServer console — one line per connected player showing their slot index and name.
SourceInline in Commands.java
Console output
[1] admin
[2] testplayer
[5] iron_man1
saveall 👑 Owner
Force-save all online players to disk immediately.
Syntax::saveall
OutputServer console logs each player saved. Chat confirms completion.
Sourcecommands/owner/Saveall.java
Example
saveall
Saved all players.
Run this before making risky server-side changes so no progress is lost if something crashes.

📦 Item Spawning 3 commands

spawn [item-id] [amount] 👑 Owner
Spawn items directly into your inventory. Supports single items, quantities, or ranges.
Syntax::spawn <item-id> <amount>
::spawn <start-id>-<end-id> <amount>
Sourcecommands/owner/Spawn.java
Examples
spawn 4151 1
Spawn 1 Abyssal whip (ID 4151)
spawn 995 10000000
Spawn 10M coins
spawn 13307 500
Spawn 500 Blood Money for testing
spawn 4151-4160 1
Spawn one of every item with IDs 4151 through 4160
giveitem [player] [item-id] [amount] ⚙ Admin
Give items to another online player. Logged to giveitem logs and posted to Discord.
Syntax::giveitem <player> <item-id> <amount>
Sourcecommands/admin/Giveitem.java
Examples
giveitem testplayer 4151 1
Give testplayer 1 Abyssal whip
giveitem testplayer 995 500000
Give testplayer 500k coins
All give actions are automatically logged — GiveItemLogger writes to C:/xampp/htdocs/logs/giveitem/ and posts to the Discord #giveitem-logs channel.
takeitem [player] [item-id] [amount] ⚙ Admin
Remove items from another online player's inventory.
Syntax::takeitem <player> <item-id> <amount>
Sourcecommands/admin/Takeitem.java
Example
takeitem testplayer 13307 1000
Remove 1000 Blood Money from testplayer

👾 NPC Tools 4 commands

npc [npc-id] 👑 Owner
Spawn an NPC at your current location. Useful for checking appearances and testing interactions.
Syntax::npc <id>
Sourcecommands/owner/Npc.java
Examples
npc 4321
Spawn the Renegade Knight (Blood Money NPC)
npc 2662
Spawn the PVM Points NPC
npc 50
Spawn NPC ID 50 to check its appearance
Spawned NPCs are not persistent — they disappear on server restart. Use npc_spawns.json for permanent spawns.
killnpc [npc-id or "all"] ⚙ Admin
Kill nearby NPCs by ID, or kill all NPCs in the area at once.
Syntax::killnpc <id> or ::killnpc all
Sourcecommands/admin/Killnpc.java
Examples
killnpc 50
Kill all nearby NPCs with ID 50
killnpc all
Kill every NPC in the surrounding area
pnpc [npc-id] ⚙ Admin
Change your player's appearance to look like an NPC — good for visual testing.
Syntax::pnpc <id>
Sourcecommands/admin/Pnpc.java
Examples
pnpc 3029
Appear as Zulrah
pnpc 0
Reset to default player appearance
Use ::unpc (owner) to reset your appearance if ::pnpc 0 doesn't work.
droptable [npc-id] [trials] 👑 Owner
Simulate an NPC's drop table and fill your bank with the results — useful for verifying drop rates.
Syntax::droptable <npc-id> <trials>
SourceInline in Commands.java
Examples
droptable 239 1000
Simulate 1000 kills of NPC 239, fill bank with results
This clears your bank before filling it with results
Clears your bank before running. Do not use on a character with items you need to keep.

📋 Logging System Reference

PlayerLogging ⚙ Automatic
Automatic per-player logging to disk. No command needed — runs in the background for all players.
Log path./data/logs/[A-Z]/[PlayerName]/[LogType].txt
Sourceutil/log/PlayerLogging.java
Log types written automatically
COMMAND — every :: command typed
TRADE_LOG — all trades
SHOP_LOG — all shop purchases
DROP_ITEM — items dropped on ground
DEATH_LOG / DIED_LOG — deaths and kills
STAKE_LOG — duel arena stakes
CHANGE_PASSWORD — password changes
CHANGE_IP_ADDRESS — IP changes on login
DONATION_TRANSACTION — donation claims
ALCH — high-alch logs
All writes are async (threaded) — logging does not block the game tick.
