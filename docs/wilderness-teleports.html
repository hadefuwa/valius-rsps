Wilderness Teleports

Overview

By default, RuneScape private servers block teleports above a set wilderness depth to enforce risk-vs-reward in the wilderness. On Valius this restriction has been removed â€” players can teleport freely from any wilderness level using any method.

Teleblock is a separate mechanic and remains fully functional. A player hit by a Teleblock spell cannot teleport until the block expires, regardless of wilderness level.

Vanilla Behaviour

The original Valius source enforced the following level caps. Above these depths, the server rejected the teleport and sent an error message to the player.

Teleport Type Cap (vanilla) Error Message
All standard teleports (spell, tablet, interface) Level 20 "You can't teleport above level 20 in the wilderness."
Amulet of Glory Level 30 "You can't teleport above level 30 in the wilderness."
Wilderness Obelisk Level 30 N/A â€” destination-only, no block on use
Portal teleports (teleport interface) Level 20 "You must be below lvl 20 wilderness to teleport"

Two exceptions existed in the vanilla code:

  • Players with the Owner rank could teleport from any depth with no restriction.
  • Players with the Mini Porazdir pet summoned bypassed the standard level‑20 cap.

Current Config â€” Unrestricted

Wilderness teleport restrictions are disabled. All players can teleport from any wilderness level using any teleport method.

The threshold constant NO_TELEPORT_WILD_LEVEL in Config.java has been raised to 99. Because the wilderness never exceeds approximately level 56, this value is never reached and all level-based checks are effectively bypassed.

In addition, the explicit inWild() check blocks in canTeleport() and startTeleport() have been removed, and the hardcoded wildLevel > 20 guard in PortalTeleports.canTeleport() has been removed.

Teleblock

Teleblock is enforced separately and is not affected by these changes. The Teleblock check reads System.currentTimeMillis() - c.teleBlockDelay < c.teleBlockLength and lives in both canTeleport() and startTeleport() â€” it was left intact when the wilderness checks were removed.

A teleblocked player cannot teleport regardless of wilderness depth or teleport method. Only the wilderness level restriction has been lifted.

The remaining references to NO_TELEPORT_WILD_LEVEL in ItemOptionTwo.java, OperateItem.java, and ClickingButtons.java use the constant from Config.java and are covered by the value change to 99.

Reverting

To restore the original vanilla restrictions:

  • In Config.java, set NO_TELEPORT_WILD_LEVEL = 20.
  • In PlayerAssistant.canTeleport(), re-add the inWild() block before the Teleblock check â€” blocking standard teleports above level 20 and glory above level 30.
  • In PlayerAssistant.startTeleport(), re-add the inWild() block before the Teleblock check â€” same logic as above, with the Owner rank and Mini Porazdir pet exemptions.
  • In PortalTeleports.canTeleport(), re-add the wildLevel > 20 guard at the top of the method.
Developer note: The level threshold is entirely controlled by NO_TELEPORT_WILD_LEVEL in Config.java for most teleport paths. Only PortalTeleports.java used a hardcoded value and required a direct code edit. The glory/obelisk cap of level 30 was also hardcoded in PlayerAssistant.java rather than reading from Config.
